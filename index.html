What's the Problem?

  • Large list will create lots of DOM Nodes
  • Slows down initial rendering by browsers
  • May cause UI freezes when scrolling
  • User experience will deteriote

4 ways to render larger lists

  • Pagination
  • Infinite scroll
  • react-virtualized
  • react-window

Pagination

  • It allows you to render data in pages
  • You control the amount of data that is shown on the page
  • DOM tree is not stressed

    • Infinite scroll

  • On initial page load, only a subset of data is loaded
  • As you scroll down the page, more data is appended
  • Google Music's artist list, Facebook timeline, Twitter's live feed

    • react-virtualized

  • Specifically designed for rendering large lists and tabular data
  • Uses a technique similar to infinite scroll called windowing
  • With windowing, only the visible parts of a list are rendered to the screen

    • react-window

  • It is complete rewrite of react-virtualized
  • It aims to address shortcomings related to size and speed
  • Covers more edge cases than react-virtualized

    • Size comparsions

    Windowing...

    • It is the concept of only rendering what is visible to the user
    • Rendered elements is a very small subset of entire list
    • "window" of visible content moves when the user continues to scroll
    • DOM nodes that exit the "window" are recycled
    • Improves both the rendering and scrolling performance of the list.

    DOM recycling

    • DOM nodes are cheap, but not free.
    • Bigger DOM === Expensive process.
    • Low DOM count === Faster process
    • Reuse already created DOM elements that are off-screen instead of creating new ones
    https://developers.google.com/web/updates/2016/07/infinite-scroller
